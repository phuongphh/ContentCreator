# Issue #16

[Bug] Pipeline summary truncated at end of report — incomplete Telegram message

## Bug Report: Pipeline Summary Truncated at End of Report

**Observed on:** 2026-04-02 at 17:33 (GMT+7)  
**Bot:** Bé MC (ContentCreator)  
**Channel:** Telegram

---

## Problem Description

The pipeline summary message generated by the ContentCreator bot is being **truncated at the end**, resulting in an incomplete report. Users receive a partial summary without knowing the report is cut off, which causes confusion and impedes troubleshooting.

**Example from 2026-04-02 pipeline run:**
> PIPELINE SUMMARY — 02/04/2026 showed a warning that video creation failed with error , but the message ended abruptly — missing the closing section, recommendations, or next steps.

---

## Impact

| Area | Impact |
|------|--------|
| User Experience | Incomplete reports erode trust and cause confusion |
| Debugging | Missing tail content hides root cause details |
| Decision Making | Users can't act on incomplete pipeline results |

---

## Steps to Reproduce

1. Run the ContentCreator pipeline under conditions that trigger a failure (e.g., short video creation)
2. Wait for the pipeline summary to be posted to Telegram
3. Observe that the summary message ends abruptly — no closing section, no complete error context
4. Compare with expected full summary format

---

## Expected Behavior

- Pipeline summary is **always delivered in full**
- If the summary exceeds Telegram's 4096-character message limit, it is **split across multiple messages** with continuation markers (e.g., , )
- All sections are present: header, body, error details, recommendations, and footer
- No silent truncation

---

## Actual Behavior

- Summary message ends mid-content, cutting off the error detail / recommendation section
- No indication to the user that the message is incomplete
- Appears to happen specifically when pipeline contains failure events (longer error context)

---

## Root Cause Hypothesis

Likely candidates (to be confirmed during investigation):

1. **Telegram 4096-char limit hit** — no message-splitting logic implemented
2. **Hardcoded string truncation** —  /  /  applied to summary buffer
3. **Exception during summary assembly** — error thrown mid-render causes partial output to be sent
4. **Markdown/formatting escape issue** — special characters break message boundary detection

---

## Acceptance Criteria

- [ ] Pipeline summaries are always delivered in full to Telegram
- [ ] Messages exceeding 4096 characters are split with clear continuation markers (e.g., , )
- [ ] Failure summaries include complete error details — no truncation on error sections
- [ ] Summary includes all standard sections: header, status, error details, recommendations, footer
- [ ] Unit/integration tests added for summary generation covering: short summary, long summary (>4096 chars), summary with failure events
- [ ] No regression on existing summary formats
